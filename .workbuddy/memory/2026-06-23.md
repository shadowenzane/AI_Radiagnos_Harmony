## v1.4 IMA 修复 + 连通测试 + 改名换图标

### 1. 修复 IMA 知识库检索失败
- **根因**: 原 apiUrl `https://lke.cloud.tencent.com/v1/knowledge/query` 是桌面版臆造的，腾讯云 LKE 实际用的是 SSE 对话接口
- **正确接口**: `https://wss.lke.tencentcloud.com/v1/qbot/chat/sse`（POST）
- **鉴权**: bot_app_key 在请求体中传递（不是 Bearer token）；apiKey 仍走 Authorization 头
- **请求体字段**: session_id / bot_app_key / visitor_biz_id / content / stream=disable / workflow_status=disable
- **响应解析**: SSE 格式，逐行解析 `data:` 开头的 JSON，提取 reply 事件的 content + knowledge 数组
- **错误码**: 460001(Token失败)/460004(应用不存在)/460021(知识库未发布)/4505004(APPKEY无效)
- KnowledgeConfig.botId 字段语义改为"Bot App Key"（配置页 label 已对齐）

## v1.5.1 IMA 测试报错修复（2026-06-23）

### 问题：知识库测试报"HTTP 200: IMA 错误码 -1: 未知错误"
- **根因**: 腾讯 LKE SSE 响应中，reply 事件的 data 字段结构可能与预期不同。
  原代码把 `obj['data']` 当 Map 处理，但实际可能是其他类型（如 String 或 null），
  导致 `data['code']` 返回 null，最终 fallback 到 `?? -1` 和 `?? '未知错误'`。
  更关键的是：当没有 error 事件但也没有成功解析到 reply 内容时，
  原代码不会报错而是静默返回空结果，但测试方法把空结果当作失败。

- **修复**: 重写 _callTencentSse 响应解析逻辑：
  1. 先收集所有 data: 行的 JSON 对象到 allDataObjs 列表
  2. 遍历 allDataObjs，按 event 字段分类处理（error/reply/其他）
  3. data 字段类型安全检查：`obj['data'] is Map ? obj['data'] as Map : obj`
  4. 没有 reply 事件且 context+docs 都为空时，返回原始响应预览作为错误信息
  5. 测试内容从 '测试' 改为 '你好'（更自然，更容易触发知识库回复）

### 关键改进
- 错误信息现在会包含原始响应预览，方便用户排查
- 成功时提示"收到回复"让用户确认知识库真的在工作
- 解析逻辑更健壮：data 字段可能是 Map / String / null，都安全处理
